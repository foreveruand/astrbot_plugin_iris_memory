# Iris Memory Plugin 用户功能文档

## 项目概述

**Iris Memory** 是一个智能记忆插件，让 AI 能够记住与用户的每一次互动，实现真正的个性化对话体验。

- **版本**: 1.9.1
- **平台**: AstrBot Plugin

---

## 第一章 记忆管理

### 1.1 手动保存记忆

用户可以主动要求 AI 记住特定信息。

**使用方式**：
```
/memory_save 我喜欢喝拿铁，不要加糖
```

**功能说明**：
- 支持保存任意类型的信息（偏好、事实、关系等）
- 自动识别记忆类型和敏感度
- 自动计算记忆重要性

**保存结果示例**：
```
✅ 记忆已保存
类型: preference
置信度: 0.85
```

### 1.2 搜索记忆

用户可以搜索已保存的记忆。

**使用方式**：
```
/memory_search 咖啡
```

**搜索结果示例**：
```
📋 找到 3 条相关记忆：

1. [preference] 我喜欢喝拿铁，不要加糖
   置信度: 0.85 | 保存于: 2024-01-15

2. [fact] 我每天早上 8 点喝咖啡
   置信度: 0.70 | 保存于: 2024-01-10

3. [interaction] 用户推荐了一家咖啡店
   置信度: 0.60 | 保存于: 2024-01-08
```

### 1.3 查看记忆统计

查看当前会话的记忆使用情况。

**使用方式**：
```
/memory_stats
```

**统计结果示例**：
```
📊 记忆统计

当前会话:
- 工作记忆: 5 条
- 情景记忆: 23 条
- 语义记忆: 8 条

记忆类型分布:
- 事实类: 12 条
- 偏好类: 8 条
- 情感类: 5 条
- 关系类: 3 条

存储空间:
- 向量数据库: 156 条记忆
- 缓存命中率: 78%
```

### 1.4 删除记忆

支持多种范围的记忆删除。

**使用方式**：
```
/memory_clear              # 清除当前会话的工作记忆
/memory_delete current     # 同上
/memory_delete user        # 删除当前用户的所有记忆
/memory_delete group       # 删除当前群的所有记忆
/memory_delete all         # 删除所有记忆（需管理员权限）
```

**删除确认**：
```
⚠️ 即将删除当前用户的所有记忆（共 36 条）
此操作不可恢复，请确认：
/memory_delete user confirm
```

### 1.5 重置系统

完全重置 Iris Memory 所有数据。

**使用方式**：
```
/iris_reset confirm
```

**警告**：此操作将删除所有记忆、画像、知识图谱数据，不可恢复。

## 第二章 智能对话

### 2.1 自动记忆捕获

AI 会自动识别并记住重要信息，无需用户手动保存。

**自动捕获触发器**：

| 触发词示例 | 记忆类型 |
|-----------|---------|
| 记住、别忘了、重要的是 | 显式记忆 |
| 我喜欢、我讨厌、我偏好 | 偏好记忆 |
| 我觉得、我感到、我很 | 情感记忆 |
| 我们是、你对我来说 | 关系记忆 |
| 我是、我有、我在 | 事实记忆 |
| 不要、不想、别 | 边界记忆 |

**智能过滤**：
- 自动过滤闲聊内容（"你好"、"谢谢"等）
- 避免重复记录相似信息
- 自动检测并处理矛盾信息

### 2.2 记忆检索与注入

AI 在对话时会自动检索相关记忆，实现个性化回复。

**检索示例**：
```
用户: 我今天想喝点什么
AI: 根据你的偏好，推荐拿铁（不加糖），或者你想尝试新的口味？
```

**检索策略**：
- **语义匹配**：理解用户意图，找到相关记忆
- **时间感知**：优先使用较新的记忆
- **情感感知**：根据用户当前情绪调整检索结果
- **混合检索**：综合多维度进行智能匹配

### 2.3 三层记忆体系

| 记忆层 | 生命周期 | 示例 |
|--------|---------|------|
| **工作记忆** | 当前会话 | "刚才说的那部电影" |
| **情景记忆** | 数周到数月 | "上周推荐的那家餐厅" |
| **语义记忆** | 永久 | "用户是素食主义者" |

**自动升级机制**：
- 工作记忆被多次访问 → 升级为情景记忆
- 情景记忆被反复确认 → 提取为语义记忆

### 2.4 记忆质量评估

每条记忆都有质量评分：

| 质量等级 | 说明 | 置信度 |
|---------|------|--------|
| 已确认 | 用户明确确认 | 90-100% |
| 高置信 | 多次提及且一致 | 75-90% |
| 中等 | 提及过但未验证 | 50-75% |
| 低置信 | 推测或间接获取 | 30-50% |
| 不确定 | 高度不确定 | 0-30% |

## 第三章 用户画像

### 3.1 自动画像构建

系统自动从对话中学习用户特征，构建多维度画像。

**画像维度**：

| 维度 | 包含内容 |
|------|---------|
| **工作** | 工作风格、目标、挑战、偏好 |
| **生活** | 生活方式、兴趣、习惯 |
| **情感** | 情感基线、触发因素、安抚方式 |
| **关系** | 社交风格、信任等级、亲密程度 |
| **人格** | Big Five 人格特质 |
| **沟通** | 正式程度、直接程度、幽默感 |

### 3.2 画像应用

**个性化回复风格**：
- 正式用户 → 使用规范语言
- 幽默用户 → 适当使用轻松语气
- 直接用户 → 简洁回复

**情感适配**：
- 检测用户情绪状态
- 负面情绪时避免提及过于积极的内容
- 提供适当的情感支持

**兴趣推荐**：
- 基于兴趣权重推荐内容
- 避免用户不感兴趣的话题

### 3.3 画像查看与更新

画像信息可通过 Web 界面查看：
- 各维度得分可视化
- 变更历史追踪
- 证据来源记录

## 第四章 主动回复

### 4.1 功能概述

AI 可以在群聊中主动发起对话，而非仅被动响应。

**触发场景**：
- 用户提到感兴趣的话题
- 情感信号检测（用户情绪低落）
- 跟进之前的对话
- 分享有价值的信息

### 4.2 开关控制

**使用方式**：
```
/proactive_reply on       # 开启当前群的主动回复
/proactive_reply off      # 关闭当前群的主动回复
/proactive_reply status   # 查看当前状态
/proactive_reply list     # 查看所有群的设置
```

**白名单模式**：
- 开启后，只有在白名单中的群才会主动回复
- 默认所有群都开启

### 4.3 静音时段

可配置静音时段，在指定时间范围内不主动回复：

**默认配置**：23:00 - 07:00

### 4.4 回复模式

| 模式 | 说明 |
|------|------|
| `rule` | 基于规则的回复判断 |
| `llm` | LLM 智能判断回复时机 |
| `hybrid` | 规则 + LLM 混合模式 |

## 第五章 群聊管理

### 5.1 群冷却模式

让 AI 在群聊中暂时"安静"，仅响应@消息和指令。

**使用方式**：
```
/cooldown          # 激活 20 分钟冷却（默认）
/cooldown 30       # 激活 30 分钟冷却
/cooldown 1h       # 激活 1 小时冷却
/cooldown status   # 查看冷却状态
/cooldown cancel   # 取消冷却
```

**冷却结果**：
```
⏸️ 已进入冷却模式（30分钟）
原因：用户请求
期间我将暂停主动回复，仅响应@消息和指令
```

**LLM 工具调用**：
AI 可以自动判断并激活冷却模式：
- 群聊过于活跃时
- 深夜时段
- 用户需要专注时间

### 5.2 群活跃度追踪

追踪群聊活跃度，动态调整行为参数。

**使用方式**：
```
/activity_status       # 查看当前群活跃度
/activity_status all   # 查看所有群的活跃度（需管理员权限）
```

**活跃度等级**：
| 等级 | 消息频率 | 行为调整 |
|------|---------|---------|
| 高 | > 100 条/小时 | 减少主动回复频率 |
| 中 | 20-100 条/小时 | 正常行为 |
| 低 | < 20 条/小时 | 适度增加主动回复 |

### 5.3 成员身份识别

在群聊中自动识别和标记成员：

**功能**：
- 解析 @ 成员的身份信息
- 关联成员昵称与用户 ID
- 在记忆中记录发送者信息

## 第十章 隐私与安全

### 10.1 敏感度分级

系统自动识别敏感信息并分级保护：

| 等级 | 说明 | 处理方式 |
|------|------|----------|
| 公开 | 无隐私风险 | 正常存储 |
| 个人 | 轻微隐私 | 加密存储 |
| 私人 | 需要保护 | 加密存储，访问控制 |
| 敏感 | 严格保护 | 严格加密，审计日志 |
| 极度敏感 | 最高保护 | 不持久化 |

### 10.2 会话隔离

**私聊隔离**：
- 每个用户的私聊记忆完全独立
- 不同用户之间无法互相访问

**群聊隔离**：
- 不同群的记忆完全隔离
- 群内记忆仅在该群可见

**跨场景隔离**：
- 私聊记忆不会出现在群聊
- 群聊记忆不会出现在私聊

### 10.3 数据控制

**用户权利**：
- 查看自己的所有记忆
- 删除自己的记忆
- 导出自己的数据

**数据保留**：
- 工作记忆：会话结束后清理
- 情景记忆：基于 RIF 评分动态管理
- 语义记忆：永久保存，用户可删除

## 第十一章 高级功能

### 11.1 多人格支持

支持为不同的 Bot 人格维护独立的记忆空间。

**应用场景**：
- 同一群有多个 Bot 角色
- 不同人格有不同的记忆和关系

### 11.2 记忆冲突处理

当检测到矛盾信息时：

**自动处理**：
- 时间戳优先：新信息覆盖旧信息
- 置信度优先：高置信信息优先
- 来源优先：用户确认的信息优先

**LLM 辅助**：
- 语义级冲突识别
- 智能合并建议
- 用户确认机制

### 11.3 记忆衰减

基于科学遗忘曲线的记忆管理：

**衰减因素**：
- 时间：记忆随时间自然衰减
- 访问：频繁访问的记忆保持更久
- 重要性：重要记忆衰减更慢
- 情感：高情感权重记忆衰减更慢

**半衰期配置**：
| 记忆类型 | 半衰期 |
|---------|--------|
| 兴趣 | 30 天 |
| 习惯 | 90 天 |
| 人格 | 365 天 |
| 价值观 | 730 天 |

### 11.4 LLM 增强

| 功能 | 说明 |

| 触发器检测 | 语义级意图理解 |

## 附录

### A. 快速开始

1. 安装插件到 AstrBot
2. 重启 AstrBot
3. 开始对话，系统自动记忆
4. 使用 `/memory_stats` 查看记忆情况
5. 访问 Web 界面管理记忆

### B. 常见问题

**Q: 记忆会占用多少空间？**
A: 每条记忆约 1-2KB，1000 条记忆约 1-2MB。

**Q: 记忆会永久保存吗？**
A: 语义记忆永久保存，情景记忆基于 RIF 评分动态管理，工作记忆会话结束清理。

**Q: 如何备份记忆？**
A: 通过 Web 界面导出数据，或备份 `data/iris_memory/` 目录。

**Q: 多个群的记忆会混淆吗？**
A: 不会，每个群的记忆完全隔离。
